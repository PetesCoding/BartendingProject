# IBA Cocktail Bar

A beautifully simple full-stack web application and REST API for browsing, searching, and managing official IBA (International Bartenders Association) cocktails.

This project makes it easy to discover cocktails by name, ingredient, or category — and includes a Virtual Bar where you add the ingredients you have and instantly see what drinks you can make.

## Features

- Browse all official IBA cocktails by category
- Search cocktails by name or ingredient
- Virtual Bar: add the ingredients you own and find cocktails you can make
- REST API for programmatic access to cocktail data
- Small, portable SQLite database ready for local development

## Demo

Run locally (instructions below) to view the web UI and API. The UI includes pages for browsing, searching, and managing your virtual bar.

## Tech Stack

- Backend: Python, Flask
- ORM: SQLAlchemy
- Database: SQLite
- Frontend: Jinja2 templates, HTML/CSS
- Scraping: BeautifulSoup4 (used to build the initial dataset)

## Getting Started

Prerequisites

- Python 3.10+ (or a compatible Python 3.x)
- pip

Quick setup

```bash
git clone https://github.com/yourusername/bartending-project.git
cd bartending-project
python -m venv .venv
source .venv/bin/activate   # macOS / Linux
.venv\\Scripts\\activate     # Windows (PowerShell)
pip install -r CocktailAPI/requirements.txt
```

Note: This repository contains two related apps: the backend API in `CocktailAPI/` and a web frontend in `CocktailWebApp/`. Each folder contains its own `requirements.txt` and entrypoints.

## Running the App

1. Build or load the database (from CSV):

```bash
python CocktailAPI/build.py
```

2. Run the API server (development):

```bash
cd CocktailAPI
flask run --host=0.0.0.0 --port=5000
```

3. Run the web frontend (development):

```bash
cd CocktailWebApp
flask run --host=0.0.0.0 --port=5001
```

Visit `http://localhost:5001` to use the web UI and `http://localhost:5000` for the API.

## API Endpoints

Examples (default host and port):

- `GET /cocktails` — list all cocktails
- `GET /cocktails/<name>` — single cocktail by name
- `GET /categories` — list all categories
- `GET /ingredients` — list all ingredients
- `GET /ingredients/<ingredient>` — cocktails containing ingredient

Adjust the host/port if you changed the defaults when running the Flask servers.

## Virtual Bar

The Virtual Bar is a UX feature in the web frontend that lets you save the ingredients you have on hand and instantly discover which cocktails you can make. Typical flow:

1. Open the Virtual Bar page in the web app.
2. Add ingredients from the master list.
3. The UI filters and shows cocktails that only require ingredients you own.

This is implemented in the frontend templates and backed by the cocktail/ingredient relationships in the database.

## Data Source

The dataset was generated by scraping Liquor.com using a custom BeautifulSoup script, exported to `iba-cocktails-web.csv`, and loaded into SQLite using the included build script. The CSV and SQLite files are present in both `CocktailAPI/` and `CocktailWebApp/` for convenience.
